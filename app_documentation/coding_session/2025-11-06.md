# Deployment to Vercel - November 6, 2025

## Deployment Checklist

### 1. Push Latest Code to GitHub
- [ ] Commit any pending changes
- [ ] Push to GitHub repository

### 2. Create Vercel Project
- [ ] Go to vercel.com and sign in
- [ ] Click "Add New Project"
- [ ] Import GitHub repository
- [ ] Vercel will auto-detect Next.js settings

### 3. Configure Environment Variables in Vercel
Add these in Vercel dashboard → Settings → Environment Variables:
- [ ] `NEXT_PUBLIC_SUPABASE_URL`
- [ ] `NEXT_PUBLIC_SUPABASE_ANON_KEY`
- [ ] `SUPABASE_SERVICE_ROLE_KEY`
- [ ] `NEXT_PUBLIC_SITE_URL` (use your Vercel URL)

### 4. Deploy
- [ ] Click "Deploy"
- [ ] Wait for build to complete
- [ ] Copy production URL

### 5. Update Supabase Auth Settings
In Supabase dashboard → Authentication → URL Configuration:
- [ ] Add Vercel URL to "Site URL"
- [ ] Add `https://your-app.vercel.app/auth/callback` to "Redirect URLs"
- [ ] Add `https://your-app.vercel.app/**` to "Redirect URLs"

### 6. Test Production
- [ ] Sign up new user
- [ ] Verify email
- [ ] Sign in
- [ ] Test password reset
- [ ] Test protected routes
- [ ] Sign out

### 7. Enable Automatic Deployments
- [ ] Verify Vercel auto-deploys on push to main (should be default)
